# CS-330
CS 330 Computational Graphics and Visualization
**CS 330 Portfolio Submission**

**How do I approach designing software?**

This project taught me how to break a real-world image down into basic shapes and rebuild it in 3D. Before this course, I would not have thought about a whiskey glass as two cylinders stacked together, but that is the kind of thinking this project required. I followed a step-by-step design process where I started with the simplest object (the desk plane), got it working, and then added one object at a time. Each time I added something new, I tested it before moving on. That incremental approach kept things manageable and made debugging way easier. I could apply this same method to any future software project by starting small, testing often, and building up from there.

**How do I approach developing programs?**

One of the biggest strategies I picked up was keeping my code modular. Instead of writing everything in one giant function, I split the work into separate functions like LoadSceneTextures(), DefineObjectMaterials(), and SetupSceneLights(). Each one has a clear job, so when something broke, I knew exactly where to look. Iteration played a huge role in my development too. I rarely got things right on the first try, especially with positioning objects and getting textures to load. But each failed attempt taught me something, whether it was that texture files need to be exactly 512x512 or that .jpg and .jpeg are not interchangeable in code. My approach evolved a lot from the first milestone to the final project. Early on I was just trying to get shapes to appear on screen, and by the end I was thinking about materials, lighting, and how everything fits together as a scene.

**How can computer science help me in reaching my goals?**

Computational graphics gave me a new way of thinking about how visuals are constructed digitally. Even though I work in healthcare communications, understanding how 3D environments are built opens up possibilities for things like medical visualizations or interactive training content. On the professional side, the problem-solving and debugging skills I developed in this course are transferable to any technical work. Learning to read error messages, trace issues through code, and test systematically are skills that apply far beyond graphics programming.
